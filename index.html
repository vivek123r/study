Object-Oriented Programming & Principles

What are the four pillars of OOP? Explain each. Easy ‚ñº
The four pillars are:

1. Encapsulation: Bundling data and methods into a single unit (class) and restricting access to internal details. Use private variables with public getters/setters.

2. Inheritance: Mechanism where a child class inherits properties and methods from a parent class. Promotes code reusability.

3. Polymorphism: Ability to take many forms. Types: Compile-time (method overloading) and Runtime (method overriding).

4. Abstraction: Hiding implementation details and showing only essential features. Achieved through abstract classes and interfaces.
Difference between Abstract Class and Interface? Medium ‚ñº
Abstract Class:
- Can have both abstract and concrete methods
- Can have constructors
- Can have instance variables (any access modifier)
- Supports single inheritance
- Use when classes share common behavior

Interface:
- Only abstract methods (Java 7) or default/static methods (Java 8+)
- Cannot have constructors
- Variables are public static final by default
- Supports multiple inheritance
- Use to define a contract that classes must follow
What is method overloading vs method overriding? Easy ‚ñº
Method Overloading (Compile-time Polymorphism):
- Multiple methods with same name but different parameters
- Happens in the same class
- Parameters must differ in number, type, or order

Method Overriding (Runtime Polymorphism):
- Subclass provides specific implementation of parent class method
- Same method signature (name, parameters, return type)
- Requires inheritance
Composition vs Inheritance: When to use which? Medium ‚ñº
This addresses the "is-a" versus "has-a" relationship.

Inheritance (is-a): A subclass inherits properties of its parent. Use when one object *is* a type of another (e.g., Dog is an Animal). Best for defining a general type hierarchy.

Composition (has-a): A class contains instances of other classes to gain their functionality. Use when one object *has* another (e.g., Car has an Engine). This is generally preferred for flexibility and avoiding the rigid structure of inheritance.
Explain Thread Safety, Immutability, and Synchronization in OOP. Hard ‚ñº
Thread Safety: Code is thread-safe if it can be executed concurrently by multiple threads without causing race conditions or data corruption.

Immutability: An object whose state cannot be modified after it is created. Immutable objects are inherently thread-safe as their state never changes, eliminating the need for explicit locking.

Synchronization: A mechanism (like locks or mutexes) used to control access to shared resources by multiple threads. It ensures that only one thread can access a critical section of code at a time, preventing race conditions.
Explain the Garbage Collection process and its goals. Medium ‚ñº
Garbage Collection (GC) is a form of automatic memory management that attempts to reclaim memory occupied by objects that are no longer referenced by the program (unreachable memory).

Goals: Free the programmer from manual memory deallocation, prevent memory leaks, and minimize "stop-the-world" pauses (where the application freezes during collection). GC relies on the **"Reachability"** concept to identify live objects.
Explain the SOLID principles in detail. Hard ‚ñº
SOLID is an acronym for five design principles intended to make software designs more understandable, flexible, and maintainable. 1. S - Single Responsibility Principle: A class should have only one reason to change. 2. O - Open/Closed Principle: Software entities should be open for extension but closed for modification. 3. L - Liskov Substitution Principle: Objects of a superclass should be replaceable with objects of its subclasses without breaking the application. 4. I - Interface Segregation Principle: Clients should not be forced to depend on interfaces they do not use. 5. D - Dependency Inversion Principle: High-level modules should not depend on low-level modules; both should depend on abstractions.
What are Design Patterns and name three common ones. Medium ‚ñº
Design Patterns are reusable solutions to common problems in software design. - Singleton (Creational): Ensures a class has only one instance and provides a global point of access. - Factory Method (Creational): Defines an interface for creating an object, but lets subclasses decide which class to instantiate. - Observer (Behavioral): Defines a one-to-many dependency so that when one object changes state, all its dependents are notified and updated automatically.
Differentiate between Association, Aggregation, and Composition. Medium ‚ñº
These describe relationships between objects (Classes). - Association: A weak link where one class uses another. - Aggregation (Has-A, Weak): "A contains B, but B can exist without A." (e.g., Department $\to$ Teacher). - Composition (Has-A, Strong): "A contains B, and B cannot exist without A." (e.g., Car $\to$ Engine).
Explain the difference between Stack and Heap memory in OOP. Medium ‚ñº
- Stack: Used for static memory allocation. Stores local variables, function calls, and reference variables (pointers). Fast access. Memory is managed automatically by OS (LIFO). - Heap: Used for dynamic memory allocation. Stores actual objects (instances). Slower access. Managed by the Garbage Collector.
What is Operator Overloading? Medium ‚ñº
Operator Overloading is a form of polymorphism where operators (like $+, -, \times$) are redefined to perform specific operations based on the types of the operands. This allows for defining custom behaviors for standard operators when applied to user-defined objects.
Explain the purpose of the `super` keyword. Easy ‚ñº
The `super` keyword is used inside a subclass method or constructor to refer to the immediate parent class's members (variables, methods, or constructors). It is primarily used to invoke the parent class constructor or to access an overridden method from the parent class.
What is a wrapper class? Give an example. Easy ‚ñº
A Wrapper Class is a class that encapsulates a primitive data type within an object. This is necessary in many frameworks/collections that can only handle object types. - Example: `int` (primitive) is wrapped by the `Integer` (class).
Describe the principle of "Program to an Interface, not an Implementation." Medium ‚ñº
This principle promotes flexibility and loose coupling. When you program to an interface (abstraction), you rely on the contract defined by the interface, not the specific details of a concrete class. This allows you to easily swap out implementations without changing the calling code.
Explain Covariance and Contravariance. Hard ‚ñº
These relate to type safety during polymorphism (subtyping). - Covariance: Allows a method to return a **more specific** type than its parent's method (subclass return type). - Contravariance: Allows a method to accept arguments of a **more general** type than its parent's method (superclass argument type).
What is multiple inheritance and why do languages like Java avoid it for classes? Medium ‚ñº
Multiple Inheritance allows a class to inherit from more than one direct parent class. Java avoids it for classes because it leads to the Diamond Problem, creating ambiguity about which parent's method implementation to use, making code unreliable. (It supports multiple inheritance of type via interfaces).
Explain Dependency Injection (DI) and Inversion of Control (IoC). Medium ‚ñº
- Inversion of Control (IoC): A principle where the flow of a program is inverted; the framework or runtime controls the flow, not the application code. - Dependency Injection (DI): A concrete technique to achieve IoC, where objects receive the objects they depend on (their dependencies) from an external source (usually a container) rather than creating them internally.
What are inner classes and static nested classes? Medium ‚ñº
Inner classes are associated with an instance of the outer class and can access all its members (even private ones). Static nested classes are also defined inside another class but behave like top-level classes and cannot access non-static members of the outer class.
Explain the difference between early binding (static) and late binding (dynamic). Medium ‚ñº
- Early Binding (Static): The linking of a function call to a definition is done at **compile time**. Primarily applies to method overloading. Faster execution. - Late Binding (Dynamic): The linking is deferred until **runtime**. The decision is based on the actual object type, allowing runtime polymorphism. Primarily applies to method overriding.
What is the Diamond Problem in multiple inheritance? Medium ‚ñº
The Diamond Problem occurs when a class inherits from two classes that both inherit from a common parent class, creating ambiguity about which parent's method to use. Example: Class D inherits from B and C, which both inherit from A. If A has a method, which version does D use?

Data Structures & Algorithms

Explain time complexity and Big O notation with examples. Easy ‚ñº
Time Complexity measures how execution time grows with input size (n).

Common complexities:
‚Ä¢ O(1) - Constant: Array access, hash table lookup
‚Ä¢ O(log n) - Logarithmic: Binary search
‚Ä¢ O(n) - Linear: Linear search, array traversal
‚Ä¢ O(n log n) - Log-linear: Merge sort, quicksort (average)
‚Ä¢ O(n¬≤) - Quadratic: Bubble sort, nested loops
‚Ä¢ O(2‚Åø) - Exponential: Recursive Fibonacci

Rule: Drop constants and lower-order terms. O(3n¬≤ + 5n + 2) $\to$ O(n¬≤)
Array vs Linked List - When to use which? Medium ‚ñº
Array:
‚úì Random access: O(1)
‚úì Cache-friendly (contiguous memory)
‚úó Fixed size (or expensive resize)
‚úó Insertion/deletion: O(n)
Use when: Need frequent random access, size known

Linked List:
‚úì Dynamic size
‚úì Insertion/deletion at known position: O(1)
‚úó Random access: O(n)
‚úó Extra memory for pointers
Use when: Frequent insertions/deletions, size unknown
Explain different types of trees and their use cases. Medium ‚ñº
1. Binary Tree: Each node has at most 2 children
2. Binary Search Tree (BST): Left < Root < Right. Search/Insert/Delete: O(log n) average, O(n) worst
3. AVL Tree: Self-balancing BST. Height difference $\le 1$. Guaranteed O(log n)
4. Red-Black Tree: Self-balancing with color property. Used in TreeMap, TreeSet
5. B-Tree: Multi-way tree for disk storage. Used in databases, file systems
6. Heap: Complete binary tree. Min-heap (parent < children) or Max-heap. Used in priority queues
7. Trie: Tree for strings. Each node represents a character. Used in autocomplete, spell checkers
What is hashing? Explain collision resolution techniques. Medium ‚ñº
Hashing converts a key into an index using a hash function for O(1) average-case operations.

Collision Resolution:

1. Separate Chaining: Each bucket has a linked list.
2. Open Addressing: All elements stored in array itself (Linear/Quadratic Probing or Double Hashing).
Explain BFS vs DFS. When to use each? Medium ‚ñº
BFS (Breadth-First Search): Explores level by level using **Queue**. Finds **shortest path** in unweighted graphs. Space: $O(w)$.

DFS (Depth-First Search): Explores as deep as possible using **Stack/Recursion**. Used for topological sort, cycle detection, and backtracking. Space: $O(h)$.
What is Dynamic Programming? Explain with example. Hard ‚ñº
Dynamic Programming (DP) solves complex problems by breaking them into simpler **overlapping subproblems** and storing results (memoization/tabulation) to avoid recomputation. Requires **Optimal Substructure**. Used for problems like Fibonacci, Knapsack, and Longest Common Subsequence.
Explain different sorting algorithms and their complexities. Medium ‚ñº
- Bubble/Selection/Insertion Sort: $O(n^2)$ time, $O(1)$ space. Simple, slow.
- Merge Sort: $O(n \log n)$ time, $O(n)$ space. Stable. Divide and conquer.
- Quick Sort: $O(n \log n)$ average, $O(n^2)$ worst. $O(\log n)$ space. Unstable. Partition around pivot.
Describe the three main Tree Traversal methods. Easy ‚ñº
1. In-order Traversal (Left, Root, Right): Used for BSTs to retrieve data in sorted order. 2. Pre-order Traversal (Root, Left, Right): Used to create a copy of a tree. 3. Post-order Traversal (Left, Right, Root): Used to delete a tree (children before parent).
Compare Adjacency Matrix and Adjacency List for Graph representation. Medium ‚ñº
Adjacency Matrix: 2D array. Space: $O(V^2)$. Good for **dense** graphs. Edge check: $O(1)$.

Adjacency List: Array of lists. Space: $O(V + E)$. Good for **sparse** graphs. Edge check: $O(\text{degree})$.
What is Amortized Analysis and when is it useful? Hard ‚ñº
Amortized Analysis calculates the average cost of a sequence of operations, where infrequent expensive operations are "paid for" by many inexpensive operations. It is useful when analyzing dynamic data structures like dynamic arrays (where resizing is rare and costly) or Fibonacci heaps.
Explain the Heap data structure and its time complexity for operations. Medium ‚ñº
A Heap is a complete binary tree satisfying the heap property (Min or Max). - **Operations:** Insertion: $O(\log n)$, Deletion (of root): $O(\log n)$, Peek (of root): $O(1)$. - **Use Case:** Efficient Priority Queue implementation.
What is a Trie (Prefix Tree) and where is it used? Medium ‚ñº
A Trie is a tree structure used for storing strings where common prefixes are shared among nodes. Search/Insertion is $O(L)$, where $L$ is the length of the key. - **Use Cases:** Autocomplete features, spell checkers, and IP routing tables.
Differentiate between Stack and Queue. Easy ‚ñº
- Stack (LIFO - Last In, First Out): Push/Pop from the top. Use for function calls, expression evaluation. - Queue (FIFO - First In, First Out): Enqueue at one end, Dequeue from the other. Use for job scheduling, BFS.
What is a Skip List and how does it improve Linked List performance? Hard ‚ñº
A Skip List is a probabilistic data structure built on top of multiple parallel sorted linked lists. It uses multiple "levels" of links to enable search operations in $O(\log n)$ average complexity, similar to a balanced BST.
Explain the concept of Topological Sort. Medium ‚ñº
Topological Sort is a linear ordering of vertices in a **directed acyclic graph (DAG)** such that for every directed edge $u \to v$, vertex $u$ comes before vertex $v$. Used for scheduling tasks with dependencies.
Explain the workings of Dijkstra's algorithm. Medium ‚ñº
Dijkstra's algorithm finds the **shortest paths** from a single source node to all other nodes in a graph with **non-negative edge weights**. It uses a **Min-Heap (Priority Queue)** to greedily select the unvisited node with the smallest known distance from the source.
What are the components of a Hash Table (Hash Map)? Medium ‚ñº
1. **Key** (Input data). 2. **Hash Function** (Converts key to index). 3. **Bucket/Slot** (Index in array). 4. **Value** (Data stored). 5. **Collision Resolution** (Chaining/Open Addressing).
What is the difference between $\Theta$ (Theta) and $O$ (Big O) notation? Hard ‚ñº
- $O$ **(Big O):** Represents the **Upper Bound** (Worst Case) complexity. Guarantees the running time will not be *worse* than this function. - $\Theta$ **(Theta):** Represents the **Tight Bound** (Average/Exact Case) complexity. The running time is bounded both above and below by this function.
Explain the use of a Minimum Spanning Tree (MST). Name two algorithms. Medium ‚ñº
An **MST** is a subgraph of an undirected, connected, weighted graph that connects all the vertices together, without any cycles, and with the **minimum possible total edge weight**. - **Algorithms:** **Kruskal's Algorithm** (uses DSU) and **Prim's Algorithm** (uses Priority Queue).
What is an NP-complete problem? Hard ‚ñº
**NP-complete** problems are the hardest problems in the NP complexity class (Non-deterministic Polynomial time). They are verifiable in polynomial time, and if an efficient (polynomial time) algorithm exists for *any* NP-complete problem, it exists for *all* of them. (e.g., Traveling Salesman, 3-SAT).

Database Management Systems & SQL

Explain ACID properties with real-world examples. Easy ‚ñº
A - Atomicity: All or nothing (Bank transfer fails midway, no money deducted).
C - Consistency: Database moves from one valid state to another.
I - Isolation: Concurrent transactions don't interfere (Two people booking last seat).
D - Durability: Once committed, changes persist even after system failure.
What is Normalization? Explain up to 3NF with examples. Medium ‚ñº
Normalization organizes data to reduce redundancy and improve integrity.

1NF: Atomic values (no multi-valued attributes).
2NF: In 1NF + No partial dependency (non-key attribute depends on part of composite key).
3NF: In 2NF + No transitive dependency (non-key $\to$ non-key).
Primary Key vs Foreign Key vs Unique Key vs Candidate Key? Easy ‚ñº
Primary Key: Uniquely identifies each record, cannot be NULL, only one per table.
Foreign Key: References Primary Key of another table, can be NULL, enforces referential integrity.
Unique Key: Ensures all values are unique, can have one NULL, multiple allowed.
Candidate Key: Minimal set of attributes that can uniquely identify records. One becomes Primary Key.
Explain different types of JOINs with examples. Medium ‚ñº
1. INNER JOIN: Only matching records.
2. LEFT JOIN: All from left table + matching from right.
3. RIGHT JOIN: All from right table + matching from left.
4. FULL JOIN: All records from both tables, with NULLs where there is no match.
What is an Index in a database and why is it used? Easy ‚ñº
An Index is a data structure (like a B-Tree) that speeds up data retrieval. - **Uses:** Faster `SELECT` queries, especially with `WHERE` clauses. - **Trade-off:** Indexes consume space and slow down `INSERT`, `UPDATE`, and `DELETE` operations.
Explain Transaction Isolation Levels. Hard ‚ñº
Isolation levels define how and when the changes made by one transaction become visible to others: 1. Read Uncommitted (Allows Dirty Reads). 2. Read Committed (Prevents Dirty Reads). 3. Repeatable Read (Prevents Non-repeatable Reads). 4. Serializable (Prevents all anomalies).
What is the difference between SQL and NoSQL databases? Medium ‚ñº
SQL (Relational): Fixed schemas, vertical scaling, follows **ACID** (strong consistency). Used for complex joins/financial data.

NoSQL (Non-Relational): Dynamic schemas, horizontal scaling, follows **BASE** (eventual consistency). Used for real-time data, flexible models.
Explain BCNF (Boyce-Codd Normal Form) and when it's necessary. Hard ‚ñº
BCNF is stricter than 3NF. A table is in BCNF if and only if for every non-trivial functional dependency $X \to Y$, $X$ is a **superkey** of the relation. It's necessary when 3NF fails due to overlapping candidate keys that cause redundancy.
What is Denormalization and when is it justified? Medium ‚ñº
Denormalization is intentionally introducing redundancy to improve **read performance** (faster querying) at the expense of write performance and increased data redundancy. Common in OLAP/Reporting systems where analysis speed is paramount.
What are Stored Procedures and Triggers? Medium ‚ñº
- **Stored Procedure:** A prepared SQL code segment saved in the database for execution on demand. Offers security and pre-compiled performance benefits. - **Trigger:** A stored procedure that automatically executes (fires) in response to a specific database event (`INSERT`, `UPDATE`, `DELETE`).
Difference between Clustered and Non-Clustered Indexes. Medium ‚ñº
- **Clustered Index:** Determines the **physical order** of data storage on disk. A table can have **only one** (usually the Primary Key). - **Non-Clustered Index:** Does not change the physical order. It contains the index keys and **pointers** to the data location. A table can have **many**.
Explain the three data anomalies in database design. Medium ‚ñº
1. **Insertion Anomaly:** Cannot insert data without also inserting unrelated data. 2. **Deletion Anomaly:** Deleting one piece of data unintentionally removes other, necessary data. 3. **Update Anomaly:** Modifying a data item requires updating multiple records, risking inconsistency.
What is SQL Injection and how can it be prevented? Easy ‚ñº
**SQL Injection** is a vulnerability allowing an attacker to interfere with database queries. - **Prevention:** Always use **Prepared Statements** (parameterized queries), which ensure user input is treated as data, not executable SQL code.
Differentiate between OLAP and OLTP systems. Medium ‚ñº
- **OLTP (Online Transaction Processing):** Manages day-to-day transactions (e.g., e-commerce orders). High volume of small read/write/update operations. Focus on **ACID**. - **OLAP (Online Analytical Processing):** Analyzes large historical data sets. Low volume of complex read-only queries. Focus on reporting and data warehousing.
What is a Database Schema vs Database Instance? Easy ‚ñº
- **Schema:** The **design** or structure of the database (tables, constraints, relationships). It is static. - **Instance:** The actual **content** or snapshot of the data in the database at a specific moment in time. It is dynamic.
Explain the concept of Two-Phase Commit (2PC). Hard ‚ñº
**2PC** is a protocol for distributed transactions ensuring all participating databases either **commit** or **abort** together. - **Phase 1 (Vote):** Coordinator asks participants to prepare and vote (YES/NO). - **Phase 2 (Decision):** If all vote YES, coordinator sends COMMIT. If any vote NO, coordinator sends ABORT.
What are Views in SQL and how are they used? Medium ‚ñº
A **View** is a virtual table based on the result-set of an SQL query. It acts as a security layer (showing only necessary columns/rows) and simplifies complex queries by pre-defining joins.
What is the difference between TRUNCATE, DELETE, and DROP? Medium ‚ñº
- **TRUNCATE (DDL):** Removes all rows, keeps structure, fast, cannot be rolled back. - **DELETE (DML):** Removes specific rows (or all), keeps structure, slow, can be rolled back. - **DROP (DDL):** Removes the entire table (structure, data, and indexes). Cannot be rolled back.
What is a functional dependency in the context of normalization? Medium ‚ñº
A **Functional Dependency** is a constraint where the values of one set of attributes ($A$) determine the values of another set ($B$). Notated as $A \to B$. It is the core concept used to define the normal forms.
What are the isolation levels in SQL (Recap)? Easy ‚ñº
The four standard isolation levels: **Read Uncommitted**, **Read Committed**, **Repeatable Read**, and **Serializable**. They trade off concurrency (speed) against data integrity (preventing read anomalies).

Operating Systems & Concurrency

Process vs Thread: What's the difference? Easy ‚ñº
Process: An instance of a program being executed. Has its own **separate memory space**. IPC is expensive.

Thread: A segment of a process. Threads within the same process **share the memory space**. Context switching is faster.
What is a Deadlock? List the four necessary conditions. Medium ‚ñº
A Deadlock occurs when two or more processes are blocked forever, waiting for resources held by each other.

Conditions: 1. **Mutual Exclusion**. 2. **Hold and Wait**. 3. **No Preemption**. 4. **Circular Wait**.
Explain Paging and Segmentation in Memory Management. Medium ‚ñº
Paging: Divides logical space into fixed-size **Pages** and physical memory into fixed-size **Frames**. Solves external fragmentation.

Segmentation: Divides logical space into variable-sized **Segments** (logical units like code/data).
Describe common CPU Scheduling Algorithms. Medium ‚ñº
1. **FCFS (First-Come, First-Served):** Simplest, non-preemptive.
2. **SJF (Shortest Job First):** Optimal for minimizing average wait time.
3. **Priority Scheduling:** Process with highest priority runs first (risks starvation).
4. **Round Robin:** Preemptive, uses time slices (fair for time-sharing systems).
Mutex vs Semaphore vs Monitor: Explain Synchronization Primitives. Hard ‚ñº
Mutex: A simple lock used for **Mutual Exclusion**. Only one thread can acquire it.

Semaphore: A signaling mechanism (counter). Can be used for mutual exclusion (Binary Semaphore, count=1) or resource pooling (Counting Semaphore, count>1).

Monitor: A higher-level construct combining critical section code and data, using a mutex and condition variables.
Explain Virtual Memory and its main purpose. Easy ‚ñº
**Virtual Memory** uses disk storage (Swap Space) as an extension of RAM, creating the illusion of a very large, contiguous memory space. Its main purpose is to allow programs larger than physical RAM to run and to provide memory protection between processes.
What is a System Call? Give examples. Medium ‚ñº
A **System Call** is the mechanism used by application programs to request a service from the operating system's kernel. They provide an interface between a process and the OS. - **Examples:** `read()`, `write()`, `fork()`, `exit()`.
Explain the difference between User-Level Threads and Kernel-Level Threads. Medium ‚ñº
- **User-Level Threads:** Managed by the application library. Kernel is unaware. Fast switching, but entire process blocks if one thread blocks.
- **Kernel-Level Threads:** Managed directly by the OS. Slower switching, but better concurrency (one thread blocking doesn't stop the process).
Describe the concept of Context Switching and its overhead. Easy ‚ñº
**Context Switching** is the process where the OS saves the state (context) of the current running process and loads the saved state of the next process. This process is pure **overhead** because no useful work is done, hence minimizing switches is key.
What are the differences between Monolithic and Microkernel architectures? Hard ‚ñº
- **Monolithic:** Entire OS kernel runs in a single address space. Fast but less reliable (a bug can crash the whole kernel).
- **Microkernel:** Only essential services (IPC, scheduling) run in kernel mode. Other services run as user-level processes. More reliable but slower due to message passing (IPC).
Explain the various Inter-Process Communication (IPC) mechanisms. Medium ‚ñº
IPC methods: **Pipes** (simple, one-way), **Message Passing**, and **Shared Memory** (fastest, requires synchronization). **Sockets** are used for network communication.
What is Internal Fragmentation vs External Fragmentation? Medium ‚ñº
- **Internal Fragmentation:** Unused space *within* an allocated memory block (e.g., wasted space inside a page frame).
- **External Fragmentation:** Total available memory is enough, but it is scattered in small, non-contiguous chunks, making it unusable. (Solved by Paging).
What is the concept of Thrashing in Virtual Memory? Hard ‚ñº
**Thrashing** is a condition where a process spends more time **paging** (swapping pages in and out of main memory) than **executing instructions**. It causes a drastic drop in system performance.
Explain the Banker's Algorithm for Deadlock Avoidance. Hard ‚ñº
The **Banker's Algorithm** determines if allocating resources to a requesting process will leave the system in a **safe state** (a sequence exists that can satisfy all processes without causing deadlock). It requires knowing the maximum resource needs in advance.
What is a Race Condition? How is it resolved? Medium ‚ñº
A **Race Condition** occurs when the outcome of a program depends on the unpredictable sequence of multiple threads accessing/modifying shared data. - **Resolution:** Use synchronization primitives like **Mutexes** or **Semaphores** to protect the **Critical Section** of code.
Describe the common Page Replacement Algorithms. Medium ‚ñº
1. **FIFO (First-In, First-Out):** Removes the oldest page.
2. **LRU (Least Recently Used):** Removes the page that has not been used for the longest time (generally effective).
3. **Optimal (OPT):** Removes the page that will not be used for the longest period (impossible in practice).
What are Interrupts and Traps? Medium ‚ñº
- **Interrupt:** A signal generated by **hardware** (e.g., I/O completion) to switch control to the kernel.
- **Trap (Software Interrupt):** A software-generated interrupt caused by an error (e.g., division by zero) or a deliberate request (e.g., a System Call).
Explain the concept of I/O Bound vs CPU Bound processes. Easy ‚ñº
- **I/O Bound:** A process that spends most of its time waiting for I/O operations (network, disk) to complete.
- **CPU Bound:** A process that spends most of its time executing instructions on the CPU (complex calculations).
What is Caching and Cache Coherency? Medium ‚ñº
- **Caching:** Storing frequently accessed data in a smaller, faster memory closer to the processor.
- **Cache Coherency:** Ensuring that all copies of shared data across multiple caches (in a multi-core system) are consistent and up-to-date.
What is a Kernel and what does it manage? Easy ‚ñº
The **Kernel** is the core component of the operating system. It manages and controls the system's hardware and software resources. - **Key tasks:** Process scheduling, memory management, I/O management, and security enforcement.

Computer Networks & Security

Compare OSI and TCP/IP Models. Easy ‚ñº
OSI Model (7 Layers): Theoretical, standard model.
TCP/IP Model (4/5 Layers): Practical, used in the Internet. Combines Session, Presentation, and Application layers of OSI into one Application layer.
TCP vs UDP: Explain the key differences and use cases. Medium ‚ñº
TCP: **Connection-Oriented** (3-way handshake), **Reliable** (guarantees delivery/order), slower. Use for HTTP/S, FTP.

UDP: **Connectionless**, **Unreliable** (best-effort), faster (low overhead). Use for DNS, VoIP, Gaming.
How does DNS (Domain Name System) work? Easy ‚ñº
DNS translates domain names (example.com) into IP addresses. It uses a hierarchy (Local, Root, TLD, Authoritative servers) to recursively find the IP. The result is cached for speed.
What is the difference between HTTP and HTTPS? Easy ‚ñº
HTTP: Data is sent in plain text.

HTTPS: Data is encrypted using **TLS/SSL**. Ensures **Confidentiality**, **Integrity**, and **Authentication**.
Explain the high-level steps when typing a URL in a browser. Hard ‚ñº
1. **DNS Lookup** (URL to IP).
2. **TCP Handshake** (Connection setup).
3. **TLS/SSL Handshake** (Security setup, if HTTPS).
4. **HTTP Request** (GET request sent).
5. **Server Processing** (Response generated).
6. **Rendering** (Browser builds page).
What is a JWT (JSON Web Token) and how is it used for authentication? Medium ‚ñº
A JWT is a digitally signed, stateless token containing user claims. The client sends the JWT in the Authorization header. The server validates the signature without a database lookup, ensuring stateless and secure API communication.
Explain the 3-Way Handshake in TCP connection establishment. Medium ‚ñº
1. **SYN:** Client sends synchronization request to server.
2. **SYN-ACK:** Server acknowledges (ACK) the client's request and sends its own SYN.
3. **ACK:** Client acknowledges the server's SYN. Connection is established.
What is Routing? Explain the difference between static and dynamic routing. Medium ‚ñº
**Routing** is selecting the path for network traffic. - **Static:** Paths are manually configured. Simple, but inflexible. - **Dynamic:** Paths are calculated automatically using **Routing Protocols** (OSPF, BGP) based on real-time network conditions. Flexible, but complex.
What are the key differences between IPv4 and IPv6? Medium ‚ñº
- **Address Size:** IPv4 is **32-bit**, IPv6 is **128-bit** (solves exhaustion). - **Security:** IPv6 has native support for **IPSec** (encryption/authentication). - **Notation:** IPv4 uses dotted decimal; IPv6 uses hexadecimal.
What is Network Latency and Bandwidth? Easy ‚ñº
- **Bandwidth:** The maximum data rate (bits per second). The "width" of the pipe.
- **Latency:** The time delay for a bit of data to travel end-to-end. The "time" it takes to cross the pipe.
Explain the purpose of NAT (Network Address Translation). Medium ‚ñº
**NAT** maps many private IP addresses (internal network) to a few public IP addresses (internet). This conserves public IPv4 addresses and adds a basic layer of security by hiding internal host structures.
What are Socket APIs and Socket Programming? Medium ‚ñº
**Socket APIs** are the programming interfaces for networking. A **Socket** is an endpoint for network communication (IP address + Port Number). **Socket Programming** is writing applications that use these interfaces for network exchange.
How does TCP handle Congestion Control? Hard ‚ñº
TCP regulates the data rate to prevent network overload using the **Congestion Window (cwnd)**. It employs algorithms like **Slow Start** (exponential increase) and **Congestion Avoidance** (linear increase/halving window size upon packet loss).
Differentiate between Hub, Switch, and Router. Medium ‚ñº
- **Hub (Layer 1):** Broadcasts to all, based on signals.
- **Switch (Layer 2):** Intelligently forwards based on **MAC addresses** (within a LAN).
- **Router (Layer 3):** Connects different networks, forwards based on **IP addresses**.
What is ARP (Address Resolution Protocol)? Easy ‚ñº
**ARP** maps an **IP address** (Layer 3) to a corresponding **MAC address** (Layer 2) within the same local network segment. Necessary for physical delivery of packets.
Explain Port Numbers and their usage. Easy ‚ñº
**Port Numbers** (16-bit) identify specific **applications or services** running on a host. They allow multiple processes to share a single IP address (e.g., HTTP: 80, HTTPS: 443).
What is a Firewall and its types? Medium ‚ñº
A **Firewall** monitors and controls incoming/outgoing network traffic based on security rules. - **Types:** Packet-Filtering (basic check), Stateful Inspection (tracks connection state), and Application-Layer (checks content/most secure).
Explain the concept of Subnetting (recap and expansion). Medium ‚ñº
**Subnetting** divides a large network into smaller subnetworks using a **Subnet Mask**. This reduces traffic (local traffic stays local), improves security, and efficiently manages IP address space.
What is DHCP (Dynamic Host Configuration Protocol)? Easy ‚ñº
**DHCP** is a protocol that automatically assigns IP addresses, subnet masks, default gateways, and other network configuration parameters to devices on a network.
What are the HTTP Response Status Codes and their classes? Easy ‚ñº
- **1xx:** Informational - **2xx:** Success (e.g., 200 OK) - **3xx:** Redirection (e.g., 301) - **4xx:** Client Error (e.g., 404 Not Found) - **5xx:** Server Error (e.g., 500 Internal Server Error)

Explain JavaScript Closures and their usefulness. Medium ‚ñº

Describe the JavaScript Prototype chain. Medium ‚ñº
Every JS object links to a **Prototype** object. If a property is not found on the object itself, the engine searches up the chain of prototypes. This mechanism is how inheritance is implemented in JavaScript.
What is CORS (Cross-Origin Resource Sharing) and how does it work? Medium ‚ñº

Explain the difference between == and === in JavaScript. Easy ‚ñº

Describe the concept of Event Bubbling and Event Capturing. Medium ‚ñº

What are Web Workers and when should you use them? Medium ‚ñº
**Web Workers** allow JavaScript code to run in a background thread separate from the main execution thread of the browser. Use them for **computationally intensive tasks** (like large data processing) to prevent the main UI thread from blocking.
Explain the use of `async`/`await` for promises. (Recap, focused on syntax). Easy ‚ñº

Difference between functional components and class components in React. Medium ‚ñº
- **Class Components:** Use ES6 classes, manage state via lifecycle methods. - **Functional Components:** Simple JS functions. Use **Hooks** (`useState`, `useEffect`) to manage state and side effects. Now the preferred standard.
How do you secure a web application against XSS (Cross-Site Scripting)? Easy ‚ñº

Explain the role of webpack or similar bundlers. Medium ‚ñº
**Bundlers** (Webpack, Parcel) analyze module dependencies and combine them into optimized static assets (bundles) for the browser. They handle transpilation (Babel), minification, and code splitting.

The **CSS Box Model** defines how elements are laid out, composed of four layers from inner to outer: **Content**, **Padding** (inside border), **Border**, and **Margin** (outside border).

System Design & Distributed Systems

What is Scalability? Differentiate Vertical vs Horizontal Scaling. Easy ‚ñº
Scalability is the ability of a system to handle increased load.

1. Vertical Scaling (Up): Adding more resources (CPU, RAM) to a single server. Limited.

2. Horizontal Scaling (Out): Adding more servers (machines). Virtually unlimited, but requires load balancing.
What is a Load Balancer and why is it essential? Medium ‚ñº
A Load Balancer distributes incoming network traffic across a group of backend servers. **Essential for:** High **Availability** (failover) and **Scalability** (distributing load).
Explain the concept of Microservices Architecture. Medium ‚ñº
Microservices breaks a large Monolith into smaller, independent services. Each runs in its own process and communicates via REST/message queues. Advantages: Independent deployment, resilience.
Describe the CAP Theorem and its implications. Medium ‚ñº
The CAP Theorem states a distributed system can only guarantee two of three: **Consistency**, **Availability**, **Partition Tolerance**. **Implication:** Since P is mandatory in large-scale systems, you must choose between **CP** (Consistency over Availability) or **AP** (Availability over Consistency).
Explain Data Sharding (Partitioning) and common keys. Hard ‚ñº
Sharding is horizontal scaling for databases, dividing data into smaller pieces (**shards**) across multiple database instances. - **Keys:** **Range-based** (e.g., User IDs 1-1000) or **Hash-based** (distributes load evenly).
What is eventual consistency and how does it relate to the BASE principles? Hard ‚ñº
Eventual Consistency: If no new updates are made, all data replicas will eventually converge to the same value. It is the trade-off chosen when prioritizing **Availability** (AP systems). - **BASE (Basically Available, Soft state, Eventually consistent)** is the set of principles guiding NoSQL systems.
What are Message Queues and when should you use them? Medium ‚ñº
**Message Queues (MQs)** decouple and asynchronously connect services. - **Use Cases:** **Decoupling** services, **Buffering/Throttling** (handling traffic spikes), and running **Asynchronous Tasks** (e.g., sending emails).
Describe the concept of Replication and its two main types. Medium ‚ñº
**Replication** is creating multiple copies of data across servers for High Availability and Read Scalability. - **Types:** **Leader-Follower (Master-Slave)** (reads from multiple, writes to one) and **Multi-Leader (Master-Master)** (writes to multiple, complex conflict resolution).
Differentiate between Latency and Throughput in system performance. Easy ‚ñº
- **Latency:** The time delay for a **single operation** to complete (measured in ms).
- **Throughput:** The **number of operations** completed per unit of time (measured in RPS).
What is a Bloom Filter and where is it used? Hard ‚ñº
A **Bloom Filter** is a probabilistic data structure used to test set membership. It may produce **false positives** (says an element is present when it's not) but **never false negatives**. Used to reduce expensive disk lookups for non-existent keys (e.g., database caches).
What is Rate Limiting and why implement it? Medium ‚ñº
**Rate Limiting** controls the number of requests a user (or IP) can make to a server over a given period. - **Purpose:** Protect the service from DoS/DDoS attacks, prevent abusive scraping, and ensure fair resource usage.
Describe the concept of Idempotency in API design. Medium ‚ñº
An operation is **idempotent** if applying it multiple times yields the same result as applying it once. Essential for handling network retries, as clients often retry timed-out requests. (`GET`, `PUT`, and `DELETE` should generally be idempotent).
How is Leader Election performed in a distributed system? Hard ‚ñº
**Leader Election** is the process by which nodes agree on a single coordinator (leader). This is typically achieved using **Consensus Algorithms** like **Raft** or **Paxos**, which ensure that even if nodes fail, the cluster can reliably elect a new leader and maintain consistent state.
Explain the difference between Push and Pull mechanisms for data transfer. Medium ‚ñº
- **Push:** Producer actively sends data to the consumer (e.g., WebSockets, server notifications). Good for low latency. - **Pull:** Consumer actively requests (polls) data from the producer (e.g., polling an API, message queue consumption). Good for flow control (consumer limits load).
What is a Caching strategy, specifically Cache-Aside vs Write-Through? Medium ‚ñº
- **Cache-Aside:** Application manages cache and database independently. (Most common). - **Write-Through:** Data is simultaneously written to both the cache and the database in one operation. Ensures consistency but adds write latency.
Why do we use Proxies (Forward and Reverse)? Medium ‚ñº
- **Forward Proxy:** Hides the **client's** identity, filters outbound traffic, caches external resources. - **Reverse Proxy:** Hides the **server's** identity, handles load balancing, SSL termination, and security before traffic reaches the backend servers.
Explain Horizontal vs Vertical Federation in databases. Hard ‚ñº
- **Horizontal Federation (Sharding):** Different **rows** go to different databases (e.g., users A-M $\to$ DB1). - **Vertical Federation:** Different **columns/tables** go to different databases (e.g., User Login info $\to$ DB1, User Profile details $\to$ DB2).
What is the difference between Consistency and Reliability in distributed systems? Medium ‚ñº
- **Consistency:** Refers to the **data state**‚Äîensuring all nodes see the same data (e.g., ACID).
- **Reliability:** Refers to the system's ability to **operate without failure** (e.g., High Availability, Fault Tolerance).
What is a CDN (Content Delivery Network)? Easy ‚ñº
A **CDN** is a geographically distributed network of servers that caches static content (images, JS) at edge locations near the user. It **reduces latency**, improves speed, and **offloads traffic** from the origin server.
What is the difference between SQL Sharding and NoSQL Partitioning? Medium ‚ñº
Conceptually the same (horizontal partitioning). **Sharding** is typically applied to SQL tables and requires explicit logic for cross-shard joins. **Partitioning** is often native to NoSQL systems and is optimized for key-based lookups, avoiding complex joins.

Advanced Coding Patterns

Describe the Sliding Window Pattern. Give a use case. Easy ‚ñº
The Sliding Window pattern uses two pointers (`start` and `end`) to efficiently process a contiguous sub-array/sub-string. It "slides" by adding an element at `end` and removing one at `start`, achieving $O(n)$ complexity. **Use Case:** Finding the Longest Substring with K Distinct Characters.
Explain the Two Pointers pattern. What are its variations? Medium ‚ñº
The Two Pointers pattern uses two indices to iterate through an array simultaneously. 1. **Opposite Ends:** Used for sorted arrays (e.g., finding a pair that sums to a target). 2. **Fast/Slow Pointers:** Used for Linked Lists (e.g., detecting a cycle).
What is Backtracking? How does it differ from DP? Hard ‚ñº
Backtracking explores all potential solutions incrementally (finding *all* solutions). If a path fails, it "backtracks" (undoes the last choice).

Dynamic Programming (DP) uses memoization to store subproblem results (finding the *optimal* solution).
Explain the Monotonic Stack/Queue pattern. Hard ‚ñº
A Monotonic Stack/Queue maintains elements in strictly increasing or decreasing order. Used to efficiently find the **Next Greater Element (NGE)** or **Previous Smaller Element (PSE)** for every element in an array in $O(n)$ time.
Explain the use of Binary Search on the Answer. Medium ‚ñº
This pattern is used when the problem asks for the *minimum possible maximum* or *maximum possible minimum* value. You apply Binary Search not on the input array, but on the **range of possible answers**, using a helper function to check feasibility at each step.
What is a Min-Heap/Max-Heap pattern? Give a typical problem type. Medium ‚ñº
Heaps are used when you need the smallest or largest element efficiently. - **Typical Problem:** **Top K Elements.** To find the 'K' largest elements, use a **Min-Heap** of size $K$. This provides $O(n \log k)$ complexity.
Describe the basic template for Breadth-First Search (BFS). Easy ‚ñº
BFS uses a **Queue** (FIFO) to explore nodes layer by layer. It requires a `visited` set to prevent cycles and often finds the shortest path in unweighted graphs.
Describe the basic template for Depth-First Search (DFS). Easy ‚ñº
DFS uses a **Stack** (LIFO) or **recursion** to explore as deep as possible before backtracking. Requires a `visited` set to prevent cycles in graphs.
Explain the Two-Heaps pattern. Hard ‚ñº
The **Two-Heaps** pattern maintains the median of a stream of numbers efficiently. It uses a **Max-Heap** for the smaller half and a **Min-Heap** for the larger half, keeping them balanced (sizes differ by at most 1). Median lookup is $O(1)$.
What is the use of a Disjoint Set Union (DSU) / Union-Find structure? Medium ‚ñº
The **DSU** manages a collection of disjoint sets. Operations: **`Find(x)`** (returns set representative) and **`Union(x, y)`** (merges sets). Used for checking connectivity (e.g., Kruskal's MST, counting connected components) with nearly $O(1)$ complexity.
What is Bit Manipulation and when is it necessary? Medium ‚ñº
**Bit Manipulation** performs operations directly on the binary representation of integers (`&`, `|`, `^`, etc.). It's necessary for high-performance low-level systems and for solving problems involving subsets, permissions, or quick parity checks.
Explain the Backtracking pattern for subset generation. Medium ‚ñº
Backtracking is ideal for generating all possible permutations, combinations, or subsets. It explores a state-space tree by making a choice, recursively calling itself, and then **undoing the choice** (backtracking) to explore the next alternative.
Describe the concept of Memoization in DP. Easy ‚ñº
**Memoization (Top-Down DP)** is the process of caching the results of expensive function calls (subproblems) and returning the cached result when the same inputs occur again. It is implemented using recursion with an added cache (hash map/array).
How does Dynamic Programming relate to recursion? Medium ‚ñº
All DP problems can be formulated recursively. DP is essentially **Recursion + Memoization (Top-Down)** or the iterative building of solutions from the base case **(Tabulation/Bottom-Up)**. The key is recognizing and efficiently solving overlapping recursive calls.
What is the K-way Merge pattern? Medium ‚ñº
The **K-way Merge** pattern efficiently merges $K$ sorted lists into a single sorted output list. It uses a **Min-Heap (Priority Queue)** to store the smallest element from each of the $K$ lists. Complexity is $O(N \log K)$.
Explain the use of a Sentinel Node (Dummy Node) in Linked Lists. Easy ‚ñº
A **Sentinel Node** is an extra, non-data node added at the beginning of a linked list. Its purpose is to simplify boundary conditions and edge cases, especially when inserting or deleting elements at the head of the list, by ensuring the list is never strictly null.
Differentiate between Greedy Algorithms and Dynamic Programming. Medium ‚ñº
- **Greedy:** Makes the locally optimal choice at each stage without looking ahead, hoping for a global optimum. - **Dynamic Programming:** Explores all possibilities and guarantees the global optimum by solving subproblems (requires optimal substructure).
What is an Interval Problem? Name two common solutions. Medium ‚ñº
**Interval Problems** involve finding overlaps, mergers, or scheduling based on a collection of time or numerical intervals. - **Common Solutions:** 1. **Sorting** the intervals (usually by start time). 2. **Sweep-Line** or Interval Merging techniques.
Explain the logic behind Two Pointers from Opposite Ends in a sorted array. Easy ‚ñº
Used in **sorted arrays** (e.g., Sum $A[i] + A[j] = Target$). Starting `left=0` and `right=n-1` allows $O(1)$ decision-making: if sum is too small, increment `left`; if too large, decrement `right`. This reduces complexity from $O(n^2)$ to $O(n)$.
Describe the logic of Fast and Slow Pointers (Floyd's Cycle Detection). Medium ‚ñº
Used primarily in Linked Lists. The **Fast Pointer** moves two steps at a time; the **Slow Pointer** moves one step. If the list contains a cycle, the fast pointer will eventually catch up to the slow pointer within the loop.

Cloud, Containers, and Deployment (DevOps)

What is the difference between Docker and a Virtual Machine (VM)? Easy ‚ñº
Virtual Machine (VM): Emulates a **full OS** (including kernel). Large, slow to boot, strong isolation.

Docker (Containers): Shares the **host OS kernel**. Lightweight, fast to boot, provides process isolation.
What is Kubernetes (K8s) and why is it needed? Medium ‚ñº
Kubernetes is a container orchestration platform that automates deploying, scaling, and managing containerized applications. **Needed for:** **Self-healing**, automated scaling, and service discovery for applications with hundreds of containers.
Explain Continuous Integration (CI) vs Continuous Delivery (CD). Medium ‚ñº
CI (Integration): Frequently merging code into a central repository where automated tests and builds are run (rapid feedback).

CD (Delivery): Code is automatically built, tested, and ready for *manual* deployment.
**Continuous Deployment** means code is automatically deployed to production.
What is a CDN (Content Delivery Network)? Easy ‚ñº
A CDN is a distributed network of servers that caches static content (images, JS, CSS) at edge locations near the user. **Benefit:** Reduces latency and offloads traffic from the origin server.
What is Infrastructure as Code (IaC)? Name two tools. Medium ‚ñº
**IaC** is the practice of managing infrastructure (servers, networks) using configuration files rather than manual processes. This makes infrastructure versionable and repeatable. - **Tools:** **Terraform** (cloud agnostic) and **CloudFormation** (AWS specific).
Explain the concept of Serverless Computing. Medium ‚ñº
**Serverless** abstracts server management. The cloud provider handles dynamic allocation and scaling. Developers only pay for execution time (e.g., AWS Lambda, Azure Functions).
Differentiate between IaaS, PaaS, and SaaS. Easy ‚ñº

What are Immutable Infrastructure and pets vs. cattle? Medium ‚ñº
- **Immutable Infrastructure:** Servers are **never modified** after deployment. Updates require replacing the old server with a new one. - **Pets vs. Cattle:** Servers are treated as "cattle" (disposable, easily replaceable) rather than "pets" (unique, manually maintained).
Explain the main difference between Monorepo and Multirepo Git strategies. Medium ‚ñº
- **Monorepo:** A single Git repository for *all* code/services. Simplifies dependency management and global refactoring. - **Multirepo:** Each service has its own dedicated repository. Simpler access control and clearer separation of concerns.
What is a Git Rebase and when should you avoid it? Medium ‚ñº
**Git Rebase** moves or combines a sequence of commits to a new base, creating a clean, linear history. - **Avoidance:** **Never rebase commits that have already been pushed to a shared remote branch** as it breaks history for collaborators.
Describe the concept of Observability (Metrics, Logs, Traces). Hard ‚ñº
**Observability** is understanding internal system state from external outputs (the 3 pillars): 1. **Metrics:** Numerical data over time (e.g., CPU usage, latency). 2. **Logs:** Time-stamped records of discrete events (e.g., error messages). 3. **Traces:** Records the path of a single request across multiple distributed services.
What is a Service Mesh (e.g., Istio, Linkerd)? Hard ‚ñº
A **Service Mesh** is a dedicated infrastructure layer for handling service-to-service communication in microservices. It manages traffic routing, security (mTLS), and observability without requiring application code changes (often using **sidecar proxies**).
Differentiate between Blue/Green, Canary, and Rolling Deployments. Medium ‚ñº

Fast rollback. - **Canary:** Routes a small percentage of user traffic to the new version (Canary) for testing before full rollout. Safest approach.
Explain the purpose of a Dockerfile. Easy ‚ñº
A **Dockerfile** is a text file containing instructions used to automate the creation of a **Docker Image**. Each instruction (e.g., `FROM`, `RUN`, `COPY`) creates a new layer in the image.
What is an API Gateway? Medium ‚ñº
An **API Gateway** acts as a single entry point for all client requests into a microservices system. It handles request routing, authentication, rate limiting, and composition before passing requests to the correct internal service.
Differentiate between Public, Private, and Hybrid Clouds. Easy ‚ñº
- **Public Cloud:** Services offered over the public internet (AWS, Azure). - **Private Cloud:** Computing resources used exclusively by one organization (on-premise or secure data center). - **Hybrid Cloud:** Combines public and private clouds, allowing applications and data to be shared between them.
What is Container Orchestration? Medium ‚ñº
**Container Orchestration** is the automated management of container lifecycles, including provisioning, deployment, scaling, networking, and de-provisioning. **Kubernetes** is the leading tool.
Explain the concept of Autoscaling. Medium ‚ñº
**Autoscaling** automatically adjusts the number of compute resources (servers/containers) allocated to an application in response to current load conditions (triggered by metrics like CPU utilization or latency).
Why are logs structured (JSON, key-value) in DevOps environments? Medium ‚ñº
**Structured Logging** makes logs machine-readable and easy to query across massive distributed systems. It enables automated parsing, filtering, and aggregation in centralized monitoring tools, which is difficult with unstructured (plain text) logs.
What is CI/CD Pipeline as Code? Medium ‚ñº
The practice of defining and managing the entire CI/CD pipeline configuration (stages, steps, testing logic) using scriptable files (e.g., YAML in GitHub Actions/Jenkins) stored in version control (Git). This applies **IaC** principles to the deployment process.

AI and Machine Learning Fundamentals

Difference between Supervised, Unsupervised, and Reinforcement Learning. Easy ‚ñº
Supervised: Learns from **labeled data** (input-output pairs). Used for classification/regression.
Unsupervised: Learns from **unlabeled data** to find patterns. Used for clustering/dimensionality reduction.
Reinforcement: Learns by interacting with an environment, receiving rewards (game AI/robotics).
Explain the Bias-Variance Trade-off. Medium ‚ñº
- Bias (Underfitting): Error due to overly simplistic assumptions (model misses relationships).
- Variance (Overfitting): Error due to excessive sensitivity to training data (model performs poorly on unseen data).
The goal is to balance the two for optimal generalization.
What is Overfitting and how do you prevent it? Medium ‚ñº
Overfitting occurs when a model learns the noise of the training data too well, failing to generalize. - **Prevention:** **Cross-Validation**, **Regularization (L1/L2)**, **Early Stopping**, and **Dropout** (in NNs).
Describe the role of Gradient Descent. Hard ‚ñº
Gradient Descent is the primary optimization algorithm used to train models. It iteratively adjusts the model's parameters (weights) to minimize the **Cost Function** (error) by moving in the *opposite* direction of the gradient (slope).
Explain Logistic Regression and its use case. Easy ‚ñº
**Logistic Regression** is a **classification** algorithm used to predict a discrete outcome (e.g., 0 or 1). It uses the **sigmoid function** ($\sigma(z) = \frac{1}{1 + e^{-z}}$) to map the output of a linear equation to a probability between 0 and 1.
Differentiate between Model Accuracy, Precision, and Recall. Medium ‚ñº
- **Accuracy:** Overall correctness (Correct Predictions / Total Predictions). - **Precision:** Of all predicted positives, how many were correct? (Minimizes False Positives). - **Recall:** Of all actual positives, how many were correctly predicted? (Minimizes False Negatives).
What are Activation Functions and why are they needed in Neural Networks? Medium ‚ñº
An **Activation Function** (e.g., ReLU) introduces **non-linearity** to the network, allowing it to learn complex, non-linear relationships in data. Without them, the network would only be able to model linear functions.
Explain the difference between CNNs and RNNs. Medium ‚ñº
- **CNN (Convolutional Neural Network):** Specialized for **image** data (grid-like topology). Uses convolutional layers to learn spatial features. - **RNN (Recurrent Neural Network):** Specialized for **sequential** data (text, time series). Has internal memory loops to persist information.
What is Feature Engineering? Easy ‚ñº
**Feature Engineering** is the process of using domain knowledge to select, transform, combine, and create relevant variables (features) from raw data. It is often the most critical step for improving model performance.
Describe the concept of Dimensionality Reduction. Name two techniques. Medium ‚ñº
**Dimensionality Reduction** reduces the number of input variables while retaining important information. - **Purpose:** Speeds up training, saves memory, and avoids the **"Curse of Dimensionality."** - **Techniques:** **PCA** (Principal Component Analysis) and **t-SNE**.
What is the difference between Batch Gradient Descent and Stochastic Gradient Descent? Medium ‚ñº
- **Batch GD:** Calculates the gradient using **all** training examples in each iteration. Slow but precise convergence. - **Stochastic GD (SGD):** Calculates the gradient using **only one** randomly chosen training example in each iteration. Much faster iterations, but noisier convergence.
What is NLP (Natural Language Processing)? Give a task example. Easy ‚ñº
**NLP** is the branch of AI that enables computers to understand and derive meaning from human languages. - **Task Examples:** Sentiment Analysis, Machine Translation, Text Summarization.
Explain the difference between Backpropagation and Gradient Descent. Hard ‚ñº
- **Gradient Descent:** The overall optimization *algorithm* used to minimize the cost function. - **Backpropagation:** The method (or mechanism) used to efficiently calculate the **gradients** needed for the Gradient Descent algorithm in a neural network (propagates error backward).
What is Clustering? Name two algorithms. Medium ‚ñº
**Clustering** is an **unsupervised learning** task where data points are grouped based on similarity. - **Algorithms:** **K-Means** (partitions data into $K$ spherical clusters) and **DBSCAN** (discovers clusters of arbitrary shape).
What is the F1 Score and when is it preferred over Accuracy? Medium ‚ñº
The **F1 Score** is the harmonic mean of Precision and Recall. It is preferred over Accuracy when dealing with **imbalanced datasets** to provide a better measure of the model's performance on the minority class.
What is a Hyperparameter? Give two examples. Easy ‚ñº
**Hyperparameters** are parameters whose values are set *before* the learning process begins (control the structure/process). - **Examples:** **Learning Rate**, **Number of Layers** in a Neural Network, **Number of Clusters** ($K$).
Explain the concept of Transfer Learning. Hard ‚ñº
**Transfer Learning** reuses a model developed for a task (e.g., image recognition) as the starting point for a second, related task. This involves taking a pre-trained model (e.g., VGG) and fine-tuning only the last few layers on a new, smaller dataset, saving time and data.
What is a Loss Function (Cost Function)? Name two common ones. Medium ‚ñº
A **Loss Function** quantifies the difference between the model's predicted output and the actual expected output (the error). The goal of training is to minimize this loss. - **Examples:** **Mean Squared Error (MSE)** (for regression) and **Cross-Entropy Loss** (for classification).
What is Epoch, Batch Size, and Iteration? Easy ‚ñº
- **Epoch:** One complete pass through the entire training dataset. - **Batch Size:** The number of training examples processed in one forward/backward pass. - **Iteration:** The number of batches needed to complete one epoch.
Describe two common types of Regularization (L1 and L2). Medium ‚ñº
Regularization prevents overfitting by adding a penalty to the loss function: - **L2 Regularization (Ridge):** Penalizes the squared magnitude of weights. Shrinks all weights toward zero. - **L1 Regularization (Lasso):** Penalizes the absolute value of weights. Tends to push irrelevant feature weights to exactly zero (performs feature selection).
